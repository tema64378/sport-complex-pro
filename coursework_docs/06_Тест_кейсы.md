# Тест-кейсы

## Формат

- ID — идентификатор теста
- Предусловия — что должно быть перед запуском
- Шаги — последовательность действий
- Ожидаемый результат — критерий успешности

## Набор тест-кейсов

| ID | Предусловия | Шаги | Ожидаемый результат |
|---|---|---|---|
| TC-01 | Backend и Frontend запущены | Открыть `http://localhost:3000` | Открывается главная страница без критических ошибок |
| TC-02 | Есть учетная запись администратора | Ввести корректные данные входа | Успешная авторизация, доступ к административным разделам |
| TC-03 | Есть учетная запись клиента | Войти как клиент | Доступ ограничен пользовательскими разделами |
| TC-04 | Авторизация под администратором | Перейти в «Клиенты», создать клиента | Новая запись появляется в таблице |
| TC-05 | Существующий клиент | Изменить данные клиента | Изменения сохраняются и отображаются в интерфейсе |
| TC-06 | Существующий клиент | Удалить запись клиента | Клиент удален из списка |
| TC-07 | Есть тренер и слот | Создать бронирование | Бронирование появляется в разделе «Бронирования» |
| TC-08 | Есть платежные данные | Создать запись платежа | Платеж отображается в списке с корректным статусом |
| TC-09 | Есть данные отчетов | Открыть раздел «Отчеты» и экспорт | Файл отчета формируется без ошибок |
| TC-10 | Есть токен сессии | Выполнить выход из системы | Сессия сбрасывается, защищенные разделы недоступны |
| TC-11 | Backend запущен | Выполнить `curl /api/ping` | Возвращается статус `running` |
| TC-12 | Неверные данные входа | Попытка авторизации | Отображается сообщение об ошибке без падения интерфейса |

## Критерии приемки

1. Не менее 90% тест-кейсов завершаются успешно.
2. Отсутствуют критические ошибки (невозможность входа, потеря данных, падение backend).
3. Все ключевые пользовательские сценарии выполняются в локальном режиме.
