# Диаграмма архитектуры и взаимодействия модулей

## 1 Архитектура программного комплекса

```mermaid
flowchart TB
    U["Пользователь (браузер)"] --> FE["Frontend (React + Vite)"]
    FE --> API["Backend API (Rust + Axum)"]
    API --> DB["SQLite (data.db)"]
    API --> FS["Файловая система (логи, бэкапы)"]
```

## 2 Внутренняя модульная структура backend

```mermaid
flowchart LR
    R["Router /api/*"] --> A["Auth module"]
    R --> M["Members module"]
    R --> C["Classes/Calendar module"]
    R --> B["Bookings module"]
    R --> P["Payments module"]
    R --> REP["Reports/Analytics module"]

    A --> DB[(SQLite)]
    M --> DB
    C --> DB
    B --> DB
    P --> DB
    REP --> DB
```

## 3 Диаграмма взаимодействия модулей (пример сценария)

```mermaid
sequenceDiagram
    participant FE as Frontend
    participant AUTH as Auth API
    participant BOOK as Booking API
    participant PAY as Payment API
    participant REP as Report API
    participant DB as SQLite

    FE->>AUTH: POST /api/auth/login
    AUTH->>DB: Проверка пользователя
    AUTH-->>FE: Токен и профиль

    FE->>BOOK: POST /api/bookings
    BOOK->>DB: INSERT booking
    BOOK-->>FE: Статус

    FE->>PAY: POST /api/payments
    PAY->>DB: INSERT payment
    PAY-->>FE: Статус

    FE->>REP: GET /api/reports/summary
    REP->>DB: SELECT агрегаты
    REP-->>FE: Данные отчета
```

## 4 Вывод

Архитектура построена по принципу разделения ответственности: интерфейс, бизнес-логика и хранилище данных изолированы, что упрощает сопровождение и расширение системы.
