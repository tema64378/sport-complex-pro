# Реальные диаграммы проекта

Ниже — комплект диаграмм в двух форматах:

1. `.mmd` — исходник Mermaid (можно редактировать).
2. `.svg` — готовая диаграмма для вставки в пояснительную записку и презентацию.

## Список диаграмм

1. `01_Архитектура_контекст.svg` — контекстная архитектура системы.
2. `02_Компоненты_backend.svg` — декомпозиция backend-модулей.
3. `03_ER_модель_физическая.svg` — физическая ER-модель (FK связи).
4. `04_ER_модель_расширенная.svg` — расширенная ER-модель (концептуальные связи).
5. `05_UML_UseCase.svg` — UML use case по ролям.
6. `06_UML_Class.svg` — UML class диаграмма доменных сущностей.
7. `07_UML_Sequence_Авторизация.svg` — последовательность: авторизация.
8. `08_UML_Sequence_Бронирование.svg` — последовательность: бронирование.
9. `09_UML_State_Платеж.svg` — диаграмма состояний платежа.
10. `10_Блок_схема_Авторизация.svg` — блок-схема алгоритма входа.
11. `11_Блок_схема_Бронирование.svg` — блок-схема алгоритма бронирования.
12. `12_Блок_схема_Отчеты_экспорт.svg` — блок-схема формирования отчета.
13. `13_IDEF0_Контекст_A-0.svg` — IDEF0 контекст A-0.
14. `14_IDEF0_Декомпозиция_A0.svg` — IDEF0 декомпозиция A0.
15. `15_BPMN_Процесс_обслуживания.svg` — BPMN-подобная модель процесса обслуживания.
16. `16_IDEF3_Сценарий_обслуживания.svg` — IDEF3 сценарный поток обслуживания.
17. `17_Дорожки_процесса_бронирования.svg` — диаграмма с дорожками (swimlane).

## Рекомендация по вставке в ПЗ

- В раздел «Проектирование»: 1–6.
- В раздел «Практическая часть»: 7–12.
- В раздел «Приложения»: 13–17 + SQL-схема `../16_SQL_схема_БД.sql`.
